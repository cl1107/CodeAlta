# Specs index

Current 1.0 implementation documentation is consolidated in the parent `doc/` folder. Start at [`doc/readme.md`](../readme.md) and follow the reading path there.

This folder is reserved for focused implementation specs only when a current contract needs more detail than the top-level docs should carry. A new spec must:

- describe implemented or approved behavior, not exploratory planning;
- identify the owning source projects and tests;
- link back from the relevant parent document;
- avoid product comparisons and migration notes;
- be removed or rewritten when the implementation changes.

There are no standalone active specs in this folder at the moment.
